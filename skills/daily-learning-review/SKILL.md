# 每日学习复盘

## 概述

这个 skill 用于把零散学习记录整理成便于回顾和行动的每日复盘。适合原始材料杂乱、重复、分散或不完整的场景。

## 快速开始

1. 先阅读原始笔记，再下结论。
2. 区分笔记中的事实和你的推断。
3. 先提炼最重要的收获。
4. 明确列出未解决问题，不要用猜测填空。
5. 最后给出下一步行动。
6. 用户未指定格式时，使用 [review-template.md](references/review-template.md)。

## 工作流程

### 1. 判断原始材料

- 判断笔记属于要点摘录、完整笔记、转录片段，还是资料摘抄。
- 尽早发现缺失上下文。
- 只有在缺少关键信息、确实会影响整理结果时，才追问用户。

### 2. 规整笔记

- 把相关内容归组。
- 删除明显重复的信息。
- 笔记里有专业术语时，保留原有表述。
- 保留“不确定”“待确认”这类标记。

### 3. 提炼学习价值

- 默认提炼 3 到 5 条核心收获，除非用户要求更详细。
- 把“学到了什么”和“还没搞懂什么”分开写。
- 遇到适合长期保留的知识点，建议整理为概念笔记。

### 4. 生成可执行复盘

- 用清楚直接的语言总结当天学习。
- 明确列出未解决问题。
- 推荐最小但有用的下一步行动。
- 有必要时，指出应该归档到哪里，例如 `notes/daily/` 或 `notes/concepts/`。

## 输出规则

- 不要捏造笔记中不存在的事实。
- 有推断时要明确说明。
- 优先使用简洁条目，不要写成长篇空话。
- 保留术语、接口名、模型名和文件路径。
- 如果资料不足，就明确说明这是一份部分复盘。

## 典型触发语句

适合下面这类请求：

- "整理我今天的 AI 学习笔记"
- "把这段零散学习记录总结成可复习的内容"
- "从这些笔记里提炼重点、疑问和下一步行动"
- "把今天吸收到的东西变成一篇学习复盘"

## 后续建议

- 如果某个知识点值得长期保留，创建或更新 `notes/concepts/` 下的文件。
- 如果复盘里出现了适合实践的点，可以在 `notes/projects/` 或 `roadmap/` 下补一个小计划。
- 如果用户只想做轻量整理，就不要过度总结。
