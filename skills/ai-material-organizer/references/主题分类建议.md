# 主题分类建议

整理 AI 学习资料时，优先从下面这些主题中选择 1 到 3 个主标签。

## 常见主题

- LLM 基础
- Prompt 工程
- Tool Calling
- MCP
- RAG
- Agents / 工作流
- OpenAI API
- 模型选择
- 评测与 Eval
- 监控与可靠性
- AI 应用架构
- Python 工程
- TypeScript 工程
- 后端开发
- 项目案例

## 分类原则

- 优先按“解决的问题”分类，不要只按资料形式分类。
- 一份资料如果同时覆盖很多内容，只保留最核心的 1 到 3 个主题。
- 工具文档类资料要标明它更偏“查询手册”还是“系统学习”。
- 偏新闻、更新、版本动态的资料，不要误标成长期基础资料。

## 优先级判断提示

更适合高优先级的资料通常有这些特征：

- 与当前学习目标直接相关
- 内容较新或是官方资料
- 可以直接支持当前练习或项目
- 解释清楚、结构完整、可操作性强

更适合低优先级或归档的资料通常有这些特征：

- 内容过旧
- 主题过散
- 与当前阶段关联不强
- 主要用于备查而不是系统学习
