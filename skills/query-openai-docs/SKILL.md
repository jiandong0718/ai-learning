# 查询 OpenAI 文档

## 概述

这个 skill 用于把“我想知道 OpenAI 官方文档怎么说”这类问题，转成稳定、可引用、尽量准确的查询流程。优先查官方文档，避免用二手资料替代一手来源。

## 快速开始

1. 先判断用户要查的是哪个产品或主题。
2. 优先使用 OpenAI 官方文档相关的 MCP 工具。
3. 先搜索，再抓取最相关页面。
4. 只提炼和问题直接相关的段落。
5. 对变化快的信息给出明确结论，不要凭记忆回答。
6. 回答时标明来源来自 OpenAI 官方文档。

## 工作流程

### 1. 明确查询目标

- 判断问题属于哪一类：
  - 模型选择
  - API 用法
  - 工具调用
  - 结构化输出
  - Agents SDK
  - Responses API
  - Realtime API
- 如果用户的问题过宽，先聚焦到单一主题。

### 2. 优先使用官方文档 MCP

- 优先搜索 OpenAI 官方文档对应的 MCP 服务。
- 先找最相关页面，再读取目标段落。
- 不要直接依赖记忆回答变化快的问题。

### 3. 必要时回退到官方站点检索

- 只有当 MCP 无法返回有效结果时，才回退到官方站点检索。
- 回退时仍然只使用官方域名相关资料。
- 如果没有找到明确答案，要直接说明而不是猜测。

### 4. 组织回答

- 先给结论，再给简短说明。
- 如果问题是“怎么做”，优先给步骤。
- 如果问题是“选哪个”，优先给适用场景差异。
- 如果问题是“最新版是什么”，明确说明基于官方资料得出的结论。

## 输出规则

- 不要把社区经验写成官方结论。
- 不要在没有查到官方依据时假装确定。
- 不要给过长摘录，优先提炼。
- 回答里保留产品名、接口名、模型名等关键术语。
- 涉及“最新”“当前”“推荐”这类问题时，必须以官方资料为准。

## 典型触发语句

- “帮我查一下 OpenAI 官方文档里怎么做结构化输出”
- “OpenAI 官方现在推荐哪个模型做代码任务”
- “Responses API 和 Chat Completions API 在官方文档里是怎么区分的”
- “帮我看看官方文档里 Agents SDK 怎么用”
