# AI 应用工程师学习路线

## 一、目标定义

AI 应用工程师的核心不是“会调用模型”，而是能够把模型、安全约束、外部工具、业务流程和工程系统整合成可用产品。

## 二、能力地图

### 1. LLM 基础

需要掌握：

- token、上下文窗口、采样参数、结构化输出
- 不同模型的能力边界、成本和延迟差异
- 推理模型与普通生成模型的典型使用场景
- 幻觉、稳定性、对齐和提示依赖性

### 2. Prompt 与工作流设计

需要掌握：

- 任务拆解
- 输出约束
- few-shot 示例设计
- 工具调用前后的状态管理
- 重试、降级、兜底策略

### 3. 工具接入与 MCP

需要掌握：

- API 调用基础
- function calling / tool calling
- MCP 的作用、服务配置和工具暴露方式
- 外部系统接入，例如搜索、数据库、文档、时间、文件

### 4. 后端工程能力

需要掌握：

- Python 或 TypeScript 至少一门
- Web API 开发
- 数据库存储
- 鉴权、日志、配置管理
- 部署、调试、错误追踪

### 5. RAG 与知识增强

需要掌握：

- 文档清洗与切分
- 向量检索基础
- 重排与召回评估
- 引用回传
- 知识库增量更新

### 6. Eval、监控与可靠性

需要掌握：

- 如何设计测试集
- 如何评估成功率、稳定性、延迟和成本
- 如何发现幻觉和行为漂移
- 如何加 guardrails 和人工兜底

## 三、推荐学习顺序

1. 先学 API、Prompt、结构化输出、tool calling
2. 再学 MCP、RAG、agent workflow
3. 再补评测、监控、成本优化
4. 最后用完整项目把能力串起来

## 四、12 周学习路线

### 第 1 阶段：建立基础，先把模型用对（第 1-4 周）

第 1 周：

- 理解 LLM 基本概念
- 学会调用模型 API
- 理解消息结构、system prompt、user prompt、JSON 输出

第 2 周：

- 学 prompt 基础方法
- 练习信息抽取、分类、改写、总结
- 学会让输出稳定、可校验

第 3 周：

- 学 tool calling
- 做一个“模型 + 外部工具”的小 demo
- 理解什么时候该让模型决定，什么时候该让代码决定

第 4 周：

- 学 MCP 基本概念
- 理解 skill、tool、workflow 三者关系
- 做一个带 MCP 工具调用的简单代理

阶段结果：

- 你能独立完成一个可调用外部工具的小型 AI 助手

### 第 2 阶段：把知识和工程接起来（第 5-8 周）

第 5 周：

- 学文档切分、embedding、向量检索基本原理
- 做一个本地知识问答 demo

第 6 周：

- 学 RAG 的召回、重排、引用回传
- 对比“有检索”和“无检索”的回答差异

第 7 周：

- 学后端工程基本能力
- 用 FastAPI、Express 或类似框架搭一个后端服务
- 把模型调用、日志、配置管理接起来

第 8 周：

- 学会多步骤 workflow
- 学会状态管理、重试、超时和失败兜底
- 把之前的 demo 整成一个可复用服务

阶段结果：

- 你能做一个可部署的 AI 应用后端

### 第 3 阶段：面向真实应用做可靠性（第 9-12 周）

第 9 周：

- 学评测思路
- 为你的 AI 应用设计测试样本
- 建立“输入-期望输出-评分标准”

第 10 周：

- 学成本、延迟和成功率监控
- 理解如何做 prompt 版本管理
- 学会回归测试

第 11 周：

- 做一个完整项目
- 例如：文档问答助手、客服辅助系统、会议总结系统、代码知识助手

第 12 周：

- 做项目复盘
- 写出架构、取舍、失败案例、优化方向
- 准备作品集材料

阶段结果：

- 你不只是“会调模型”，而是能解释为什么这样设计、怎么验证效果、怎么持续改进

## 五、建议输出的项目作品

至少完成 2 到 3 个项目：

1. 一个带工具调用的 AI 助手
2. 一个带 RAG 的知识问答系统
3. 一个带评测和监控的 AI 应用

## 六、判断自己是否学会了

你可以用这些问题自测：

- 我能解释这个 AI 应用为什么需要模型吗？
- 我能说明哪些部分应该交给代码，哪些部分交给模型吗？
- 我能设计失败兜底和重试逻辑吗？
- 我能证明新版本比旧版本更好吗？
- 我能把一个想法变成可运行、可观察、可迭代的系统吗？
